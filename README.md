AI 레시피 메이트 최종 기획안 (PRD 및 상세 명세)

1. 프로젝트 요구사항 정의서 (PRD 요약)

본 섹션은 초기 기획 단계에서 확정된 핵심 요구사항 및 목표를 요약합니다.

항목

내용

제품명

AI 레시피 메이트

핵심 목표

사용자가 집에 있는 재료를 입력하면, AI가 맞춤형 레시피를 즉시 생성하여 요리 고민을 줄이고 음식 낭비를 방지한다.

핵심 가치

재료 기반 레시피 자동 생성, 10분 내 간편 조리, 음식물 낭비 감소, 생활 편의 향상

MVP 범위

텍스트 기반 재료 입력 → 10분~20분 내 조리 가능한 레시피 출력 (웹 UI 기준)

AI 도구

Gemini (레시피 생성), Embedding 모델 (재료 의미 분석), Vision-Language 모델 (향후 이미지 기반 확장)

KPI 목표

생성 속도 5초 이하, 사용자 만족도 ≥80%, 레시피 완성도 ≥90%

목록 항목: 요리 이름, 생성 날짜, 난이도 요약 정보를 표시하며, 각 항목 클릭 시 메인 화면(C 영역)으로 돌아가 상세 레시피를 표시.

관리 기능: 저장된 레시피에 대한 삭제 버튼 제공.

4. UI/UX 요구사항 (UI/UX Requirements)

요리 초보자와 모바일 사용자를 위한 핵심 디자인 및 인터랙션 원칙을 정의합니다.

4.1. 디자인 원칙

모바일 우선 (Mobile First): 전체 디자인은 1~2인 가구의 모바일 사용 환경을 최우선으로 고려하여 설계. 모든 요소는 터치 친화적이어야 함.

화이트 스페이스 활용: 정보 과부하를 막고, 생성된 레시피를 읽기 쉽도록 충분한 여백(패딩 및 마진)을 확보.

색상 및 톤: 청결하고 식욕을 돋우는 밝은 계열의 색상(Food-friendly palette) 사용.

4.2. 사용자 경험 (UX) 원칙

즉각적인 피드백: 레시피 생성 버튼 클릭 시 즉시 로딩 인디케이터를 표시하여 사용자에게 '작동 중'임을 인지하게 하고, KPI인 5초 이내에 결과를 제공해야 함.

간결한 플로우: 재료 입력 → 설정 → 생성 → 결과 확인의 흐름이 단 1~2단계의 명확한 상호작용으로 끝나야 함.

입력 유연성: 사용자가 어떤 방식으로 재료를 입력하더라도(띄어쓰기 1개, 2개, 콤마 등), 시스템이 이를 유연하게 해석하고 사용자에게 피드백을 제공해야 함.

가독성: 조리 단계는 굵은 글씨, 명확한 번호 매기기, 간결한 문장으로 구성하여 조리 중에도 쉽게 따라 할 수 있도록 설계. (폰트 크기 및 대비 최적화)
